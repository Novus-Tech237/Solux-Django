Html List and Tables

Html List

To list out a number of items, The Html list offers you the opportunity to do that.

The Html list provides 3 types of list which are:

  • <ul>: Unordered List, They are used to create bullet-pointed lists. Each list item is wrapped in <li> tags
  • <ol>: Ordered List, They are used to create numbered lists. Each tem is also wrapped in <li> tags
  • <dl>: Definition List, They consist of terms and their corresponding definitions. Each term is wrapped in <dt> tags, and each definition is wrapped in <dd> tags

Example: Using <ul>

<ul>
<li>Item One</li>
<li>Item Two</li>
<li>Item Three</li>
</ul>

Example: Using <ol>

<ol>
<li>Item One</li>
<li>Item Two</li>
<li>Item Three</li>
</ol>

Example: Using <dl>

<dl>
<dt>HTML</dt>
<dd>The standard markup language for creating web pages.</dd>
<dt>CSS</dt>
<dd>A stylesheet language used for describing the presentation of a document written in HTML.</dd>
</dl>

Lists can be nested inside one another, allowing for hierarchial organization of content.

You can use CSS to customize the appearance of the list such as changing the bullet style or numbering format.

Html Tables

Tables in html are created by using the <table> element

Tables consists of table rows <tr> ,table data <td> and table headers <th>

The <thead>, <tbody> and <tfoot> are used to groupt table headers, table body content and table footes respectively.

The structure of a table is defined by placing the appropriate content table elements. That is all the elements found on the table head should be in thead tag, the table main content should be in the tbody tag and the conclusion content should be found in the tfoot

Example

<table>
<thead>
<tr>
<th>Product</th>
<th>Price</th>
</tr>
</thead>
<tbody>
<tr>
<td>Apple</td>
<td>500,0 XAF</td>
</tr>
<tr>
<td>Orange</td>
<td>200,0 XAF</td>
</tr>
</tbody>
<tfoot>
<tr>
<td>Total Price</td>
<td>700,0 XAF</td>
</tr>
</tfoot>
</table>

Remarks that on this basic code won't provide borders unless CSS is been used on the table

Note: It is important to use proper semantics and structure for both lists and tables. It enhances accessibility and allows assistive technologies to interpret the content correctly
